# Security Policy

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Report vulnerabilities privately to the maintainers with:
- A clear description of the issue
- Steps to reproduce
- Potential impact
- Suggested mitigation (if available)

## Scope

This includes frontend code, workflows, integrations, and deployment configuration.

## Response Expectations

- Initial acknowledgment: within 3 business days
- Triage and severity assessment: within 7 business days
- Fix timeline: depends on severity and complexity

## Safe Disclosure

Please allow maintainers time to investigate and patch before public disclosure.
